Accomplishments

Well Master Web Site - An Angular 1.3 web site using a C# Web API 2 backend that provides view and update capabilities for wells and related entities in a central database where information is collected and rationalized.

Cost Forecasting Desktop App - A highly complex C# WinForms application using MEFMicrosoft Extensibility Framework for modularity and DevExpress components for displaying hierarchical grids of data and supporting exports to Excel files. This application was developed in 2016 for the 2017 budget year. This application provided a huge savings in the assembly and analysis of costs related to drilling, completion and production operations in Ohio. The VP of Operations has asked for the two other department conduction field operations (Louisiana and Oklahoma) to use this application for their 2018 budgeting.

Generic, Windows Service-Based Task Processing Engine - A multi-threaded Windows Service built with C# 4.5 and leveraging MEFMicrosoft Extensibility Framework for loading pluggable modules that perform task processing functionality. This 'task processing service/engine' has been deployed in multiple instances with each instance being responsible for hosting various types of 'task processing modules'. This 'generic framework' with clean separation between the 'task processing engine' and the individual 'task processing modules' has saved significant development effort since only the 'pluggable modules' need to be built when new work is required. Modules can be built for just about any imaginable task and they all run in a completely consistent generic framework.

The generic task processing service and the over 20 task processing modules deployed so far make use of a database for scheduling, parameterization, and configuration WCFWindows Communication Framework web service requests which are processed by similar pluggable 'request processing modules'. A separate C# WinForms desktop application called the 'OpsManager' is used to create and update schedules and parameters.

Generic, WCF-Based Request Processing Web Service - A WCFWindows Communication Framework web service built with C# 4.5 and leveraging MEFMicrosoft Extensibility Framework for loading pluggable modules that perform request processing functionality. This web service has been deployed in multiple instances with each instance being responsible for hosting various types of 'request processing modules'. This 'generic framework' with clean separation between the 'web service engine' and the individual 'request processing modules' has saved significant development effort since only the 'pluggable modules' need to be built when new work is required. Modules can be built for just about any imaginable request-response type of functionality and they all run in a completely consistent generic framework.

Current Projects

Implementation of CygNet SCADA System for Ohio - I am currently heavily involved with the implementation of the CygNet SCADA System for Gulfport's Ohio assets. Of particular focus for me has been the consideration of hosting options as we currently have a number of systems on premise and others being hosted by various cloud-based technology providers. I have also been heavily invovled with infrastructure planning and implementation and ensuring that we are prepared for the systems stringent operational requirements and can also support both planned infrastructure and application maintenance and unplanned operational issues.

Text Processing Workbench - I am currently developing a C# WinForms desktop application which is a 'Text Processing Workbench' which will be used to extract raw text from PDF documents and determine, through text pattern recognition, whether the document is of a recognized type. For recognized documents, additional functionality based in a configurable command language (also developed by me) will first impose a hierarchical structure upon the text (as many hierarchical layers as desired), and then extract the desired text values from each hierarchical node and create a format-configurable output XML document. The Text Processing Workbench has been designed to allow non-programmers to develop document recognition configurations and text extraction configurations for any PDF file which will be used as a source for data collection. The workbench utilizes modular text-processing components that be embedded in other applications to automate the extraction of valuable data from PDF files and support downstream integration.
